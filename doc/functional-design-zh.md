## 核心概念

### Benchmark（评测基准集）

由 10 个以上输入样本组成的集合，每个样本包含**文本 prompt + 参考图片 + 参考视频（可选）**。

- 支持命名与版本更新（如 `Portrait-50 v2.0`）
- 每个样本有自增 ID，可通过 `{benchmark_id}_{sample_id}` 寻址（如 `portrait50_012`）
- 支持一键批量下载所有输入数据
- 通过 Web UI 进行完整增删改查操作

### 模型（Model）

已注册的视频生成模型，包含版本和类型元数据。

- 模型类型包括：AvatarIV、Tokyo、Custom 等
- 可通过 Web UI 或 API 上传某 Benchmark 上的生成结果
- 支持版本历史追踪

### 实验（Experiment）

选定多个已注册模型 + 一个 Benchmark 进行对比评测的单次评测会话。

- 评测人员逐样本对比模型输出
- 选出最佳结果（Pick Best）
- 可选填写偏好原因
- 可对每个视频打标签（如唇形不同步、画面瑕疵等）
- 若评测人员曾在其他实验中标注过同一样本，将展示历史标注记录

### 客观评分（Objective Scores）

通过统一 API 上传的自动化质量指标。

- 指标包括：唇形同步、清晰度、自然度、动作质量、身份保持、音画同步等
- 所有指标遵循统一 API 规范
- 支持按模型和按样本的对比视图

## 各页面功能规格

## 1. 登录与身份验证

### 使用场景

内部评测人员通过邮箱和密码登录，管理员负责角色与权限管理。

### 功能

- **登录**：邮箱 + 密码验证
- **角色**：Admin（管理员）、Evaluator（评测员）、Viewer（查看者）
- **权限矩阵**：

| **权限** | **Admin** | **Evaluator** | **Viewer** |
| --- | --- | --- | --- |
| CRUD Benchmarks | ✓ | ✗ | ✗ |
| CRUD 模型 | ✓ | ✗ | ✗ |
| 创建 / 管理实验 | ✓ | ✗ | ✗ |
| 参与评测（投票 / 标注） | ✓ | ✓ | ✗ |
| 查看实验结果与统计 | ✓ | ✓ | ✓ |
| 上传客观评分 | ✓ | ✗ | ✗ |
| 账号注册 / 管理 | ✓ | ✗ | ✗ |

---

## 2. 仪表盘

### 使用场景

登录后的落地页，提供平台活动的快速概览。

### 功能

- **统计卡片**（4 列网格）：活跃实验数、模型总数、Benchmark 数、本周评测数 — 每项附带趋势百分比徽章
- **近期实验表格**：实验名称、关联 Benchmark、对比模型、状态（进行中 / 已完成）、进度条、创建日期
- **动态日志**（右侧面板）：近期操作的实时日志，如"张三对样本 portrait50_012 进行了投票"、"管理员上传了 Tokyo v3.1 的结果"

---

## 3. Benchmark 管理

### 使用场景

管理员创建和维护评测数据集，评测员和查看者可浏览。

### 功能

- **列表视图**：卡片布局（2 列网格），每张卡片显示名称、版本、样本数、缩略图预览
- **搜索与筛选**：按名称搜索、按版本筛选、排序选项
- **创建 Benchmark**：设置名称、版本，批量上传样本（文本 + 图片 + 视频）
- **详情 / 展开视图**：逐一浏览样本 — 样本 ID、文本 prompt、参考图缩略图、参考视频标识
- **批量下载**：一键将所有输入数据打包下载为 zip
- **版本管理**：同一 Benchmark 可更新至新版本（如 v1.0 → v2.0）

### 数据结构

```
Benchmark:
  - benchmark_id: string（唯一标识符）
  - name: string（名称）
  - version: string（版本）
  - samples: [
      {
        sample_id: int（自增）,
        text_prompt: string（文本提示）,
        ref_image: file（必填）,
        ref_video: file（可选）
      }
    ]
```

## 4. 模型管理

### 使用场景

管理员注册模型并上传生成结果，评测员可查看模型信息。

### 功能

- **模型列表表格**：模型名称、类型徽章（AvatarIV / Tokyo / Custom — 颜色编码）、版本、已测 Benchmark 数量、已上传结果数量、最后更新时间、操作
- **注册模型**：填写模型名称、类型、版本
- **模型详情面板**（右侧）：
    - 版本历史时间线（v2.1 → v2.0 → v1.5）
    - Benchmark 结果上传状态（绿色对勾 = 已上传）
- **上传结果**：
    - Web UI：选择模型 + Benchmark，上传生成视频文件
    - API：标准接口端点，附带代码片段展示

### API 规范

```
POST /api/v1/models/{model_id}/results
Content-Type: multipart/form-data

{
  "benchmark_id": "portrait50",
  "version": "v2.0",
  "results": [
    { "sample_id": 1, "video": <file> },
    { "sample_id": 2, "video": <file> }
  ]
}
```

## 5. 评测界面（核心页面）

### 使用场景

评测员在实验中逐样本对比多个模型的视频输出，选出最佳结果并标注问题。

### 功能

#### 顶部导航栏

- 实验名称（如"AvatarIV v2.1 vs Tokyo v3.0"）
- 关联 Benchmark 名称
- 进度指示（如"已完成 12/50 个样本"）
- 上一个 / 下一个导航箭头

#### 样本信息卡片

- 样本 ID（如 `portrait50_012`）
- 完整文本 prompt 展示
- 参考图缩略图
- 参考视频缩略图（如有）

#### 视频对比区域（主体区域）

- 两个等宽视频播放器并排显示，每个置于白色圆角卡片内
- 模型标签显示为"模型 A" / "模型 B"（隐藏真实名称以减少偏见）
- 16:9 视频播放器，支持播放 / 暂停 / 进度拖拽
- 每个视频下方：
    - **"选为最佳"按钮** — 选中后变为靛蓝色并显示对勾
    - **评论文本框** — 填写针对该视频的具体反馈
    - **历史评论** — 若曾评测过，在灰色卡片中显示并附时间戳

#### 右侧面板

- **选择原因**：选定偏好后，显示文本框"为什么选这个？"，附快捷标签（更好的唇形同步 / 更自然 / 更清晰 / 动作更好）
- **问题标签**：可对每个视频打标 — 唇形不同步、动作不自然、画面模糊、画面瑕疵、身份漂移、音画不同步
- **历史标注**：若评测员在其他实验中标注过该样本，在可折叠卡片中展示

#### 底部操作

- "提交并下一个" — 主按钮，提交投票并跳转至下一个样本
- "跳过" — 次要按钮，跳过当前样本

## 6. 实验管理与结果

### 使用场景

管理员创建实验，所有用户均可查看结果和统计数据。

### 创建实验

- 选择参与模型（多选）
- 选择 Benchmark
- 设置实验名称
- 分配评测员

### 结果概览

- **胜率统计卡片**：模型 A 胜率 62%、模型 B 胜率 31%、平局/跳过 7%
- **堆叠条形图**：投票分布可视化（靛蓝 / 翠绿 / 灰色）

### 客观评分对比

- **对比表格**：行 = 指标（唇形同步、清晰度、自然度、动作质量、身份保持），列 = 模型
- 显示均值 ± 标准差
- 较优分数绿色高亮，较差分数橙色标注

### 单样本详情

- **详情表格**：样本 ID、投票胜者（模型名称或平局）、投票人、投票原因（截断显示）、各指标迷你条形图、展开按钮
- **展开视图**：
    - 双视频缩略图
    - 完整投票评论
    - 问题标签详情
    - 客观评分雷达图

### 评测员排行榜

- 侧边栏展示各评测员已完成的样本数量

## 7. 客观评分管理

### 使用场景

管理员注册客观评测指标，并通过统一 API 上传评分。

### 功能

#### 指标注册

- **指标卡片网格**（3 列）：唇形同步分、清晰度分、自然度分、动作质量、身份保持、音画相关性
- 每张卡片显示：指标名称、图标、描述、数值范围（0–1 或 0–100）、状态（启用 / 停用）
- 支持添加自定义指标

#### API 集成

统一 API 规范，附代码块展示：

```
POST /api/v1/scores/upload

{
  "model_id": "avatariv_v2.1",
  "benchmark_id": "portrait50_v2",
  "metric_name": "lip_sync",
  "scores": [
    { "sample_id": 1, "score": 0.87 },
    { "sample_id": 2, "score": 0.92 }
  ]
}

响应：
{
  "status": "success",
  "records_created": 2
}
```

#### 上传历史

- 表格显示：指标名称、模型、Benchmark、样本数、上传人、日期、状态（成功 / 失败 / 处理中）

#### 快速统计（右侧面板）

- 已注册指标总数
- 评分记录总数
- 最后上传时间

## 设计规范

### 色彩方案

| **Token** | **色值** | **用途** |
| --- | --- | --- |
| Primary | `#000000` | 主按钮、关键操作 |
| Background | `#FCFCFC` | 页面背景 |
| Surface | `#FFFFFF` | 卡片、面板 |
| Neutral | `#F3F4F6` | 次级背景 |
| Gray Text | `#9CA3AF` | 次级文字、标签 |
| Indigo | `#6366F1` | 强调色、图表、选中态 |
| Emerald | `#10B981` | 成功、正向指标 |
| Orange | `#FB923C` | 警告、注意 |

### 字体排版

- **标题**：General Sans（半粗体，字距收紧）
- **正文 / UI**：Satoshi

### 布局规范

- 固定 256px 玻璃拟态侧边栏
- 80px 固定高度 sticky 顶部栏，带背景虚化
- 主内容区 12 列网格
- 卡片圆角：32px，阴影：`0 4px 20px -4px rgba(0,0,0,0.03)`
- 按钮：完全圆角，hover 过渡 300ms ease-in-out，点击缩放 scale(0.95)
