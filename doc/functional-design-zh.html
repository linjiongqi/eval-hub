评测平台 — 功能设计文档

一个视频生成模型评测平台，支持多模型并排对比、主观偏好投票、客观评分管理及实验结果分析。

✦ 核心概念

📋 Benchmark（评测基准集）

由 10 个以上输入样本组成的集合，每个样本包含文本 prompt + 参考图片 + 参考视频（可选）。

  • 支持命名与版本更新（如 Portrait-50 v2.0）
  • 样本通过 {benchmark_id}_{sample_id} 寻址
  • 支持一键批量下载所有输入数据
  • 通过 Web UI 完整增删改查

🤖 模型（Model）

已注册的视频生成模型，包含版本和类型元数据。

  • 模型类型：AvatarIV、Tokyo、Custom 等
  • 可通过 Web UI 或 API 上传生成结果
  • 支持版本历史追踪

🧪 实验（Experiment）

选定多个已注册模型 + 一个 Benchmark 进行对比评测的单次评测会话。

  • 评测员逐样本对比模型输出
  • 选出最佳结果（Pick Best）
  • 可对每个视频打标签
  • 展示跨实验历史标注记录

📊 客观评分（Objective Scores）

通过统一 API 上传的自动化质量指标。

  • 唇形同步、清晰度、自然度、动作质量等
  • 所有指标遵循统一 API 规范
  • 支持按模型和按样本的对比视图

2 仪表盘

📈
使用场景
登录后的落地页，提供平台活动的快速概览。

功能

  • 统计卡片（4 列网格）：活跃实验数、模型总数、Benchmark 数、本周评测数 — 每项附带趋势百分比徽章
  • 近期实验表格：实验名称、关联 Benchmark、对比模型、状态（进行中 / 已完成）、进度条、创建日期
  • 动态日志（右侧面板）：近期操作的实时日志

3 Benchmark 管理

📦
使用场景
管理员创建和维护评测数据集，评测员和查看者可浏览。

功能

  • 列表视图：卡片布局（2 列网格），每张卡片显示名称、版本、样本数、缩略图预览
  • 搜索与筛选：按名称搜索、按版本筛选、排序选项
  • 创建 Benchmark：设置名称、版本，批量上传样本（文本 + 图片 + 视频）
  • 详情 / 展开视图：逐一浏览样本 — 样本 ID、文本 prompt、参考图缩略图、参考视频标识
  • 批量下载：一键将所有输入数据打包下载为 zip
  • 版本管理：同一 Benchmark 可更新至新版本（如 v1.0 → v2.0）

数据结构

Benchmark:
  - benchmark_id: string（唯一标识符）
  - name:         string（名称）
  - version:      string（版本）
  - samples: [
      {
        sample_id:   int（自增）,
        text_prompt: string（文本提示）,
        ref_image:   file（必填）,
        ref_video:   file（可选）
      }
    ]

4 模型管理

🤖
使用场景
管理员注册模型并上传生成结果，评测员可查看模型信息。

功能

  • 模型列表表格：模型名称、类型徽章（AvatarIV / Tokyo / Custom — 颜色编码）、版本、已测 Benchmark 数量、已上传结果数量、最后更新时间
  • 注册模型：填写模型名称、类型、版本
  • 模型详情面板（右侧）：
    • 版本历史时间线（v2.1 → v2.0 → v1.5）
    • Benchmark 结果上传状态（绿色对勾 = 已上传）
  • 上传结果：Web UI 上传 / API 端点（附代码片段）

上传接口

POST /api/v1/models/{model_id}/results
Content-Type: multipart/form-data

{
  "benchmark_id": "portrait50",
  "version": "v2.0",
  "results": [
    { "sample_id": 1, "video": <file> },
    { "sample_id": 2, "video": <file> }
  ]
}

顶部导航栏

  • 实验名称（如"AvatarIV v2.1 vs Tokyo v3.0"）
  • 关联 Benchmark 名称
  • 进度指示（如"已完成 12/50 个样本"）
  • 上一个 / 下一个导航箭头

视频对比区域

  • 两个等宽视频播放器并排，模型标签显示为"模型 A / 模型 B"（隐藏真实名称）
  • 16:9 播放器，支持播放 / 暂停 / 进度拖拽
  • "选为最佳"按钮 — 选中后变为靛蓝色并显示对勾
  • 评论文本框 — 填写针对该视频的具体反馈
  • 历史评论 — 若曾评测过，在灰色卡片中显示并附时间戳

右侧面板

  • 选择原因：文本框 + 快捷标签（更好的唇形同步 / 更自然 / 更清晰 / 动作更好）
  • 问题标签（每个视频独立）：唇形不同步 · 动作不自然 · 画面模糊 · 画面瑕疵 · 身份漂移 · 音画不同步
  • 历史标注：跨实验历史记录，可折叠展示

6 实验管理与结果

📊
使用场景
管理员创建实验，所有用户均可查看结果和统计数据。

创建实验

  • 选择参与模型（多选）
  • 选择 Benchmark
  • 设置实验名称
  • 分配评测员

结果概览

  • 胜率统计卡片：模型 A 胜率 62%、模型 B 胜率 31%、平局/跳过 7%
  • 堆叠条形图：投票分布可视化（靛蓝 / 翠绿 / 灰色）

客观评分对比

  • 对比表格：行 = 指标，列 = 模型，显示均值 ± 标准差
  • 较优分数绿色高亮，较差分数橙色标注

单样本详情

  • 详情表格：样本 ID · 投票胜者 · 投票人 · 投票原因（截断）· 各指标迷你条形图 · 展开按钮
  • 展开视图：双视频缩略图 + 完整投票评论 + 问题标签详情 + 客观评分雷达图
  • 评测员排行榜（右侧）：各评测员已完成样本数

7 客观评分管理

🔢
使用场景
管理员注册客观评测指标，并通过统一 API 上传评分。

指标注册

  • 指标卡片网格（3 列）：唇形同步分 · 清晰度分 · 自然度分 · 动作质量 · 身份保持 · 音画相关性
  • 每张卡片：指标名称、图标、描述、数值范围（0–1 或 0–100）、启用 / 停用
  • 支持添加自定义指标

API 集成

POST /api/v1/scores/upload

{
  "model_id":     "avatariv_v2.1",
  "benchmark_id": "portrait50_v2",
  "metric_name":  "lip_sync",
  "scores": [
    { "sample_id": 1, "score": 0.87 },
    { "sample_id": 2, "score": 0.92 }
  ]
}

// 响应
{
  "status": "success",
  "records_created": 2
}

上传历史

  • 表格：指标名称 · 模型 · Benchmark · 样本数 · 上传人 · 日期 · 成功 / 失败 / 处理中

快速统计（右侧面板）

  • 已注册指标总数
  • 评分记录总数
  • 最后上传时间
